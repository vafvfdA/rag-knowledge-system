### 组织标签授权分析
系统通过组织标签实现细粒度的权限控制。

#### 组织标签授权流程
```mermaid
flowchart TD
Start([请求进入]) --> ExtractToken["提取Token"]
ExtractToken --> GetResource["获取资源信息"]
GetResource --> CheckPublic{"资源公开?"}
CheckPublic --> |是| AllowAccess["允许访问"]
CheckPublic --> |否| CheckOwner{"用户是拥有者?"}
CheckOwner --> |是| AllowAccess
CheckOwner --> |否| CheckAdmin{"用户是管理员?"}
CheckAdmin --> |是| AllowAccess
CheckAdmin --> |否| CheckPrivate{"私人资源?"}
CheckPrivate --> |是| DenyAccess["拒绝访问"]
CheckPrivate --> |否| GetUserTags["获取用户组织标签"]
GetUserTags --> CheckMatch{"标签匹配?"}
CheckMatch --> |是| AllowAccess
CheckMatch --> |否| DenyAccess
AllowAccess --> End([放行请求])
DenyAccess --> End
```

## 性能考虑
系统通过多种机制优化性能：

1. **Redis缓存**：使用Redis缓存JWT令牌状态和组织标签信息，减少数据库查询
2. **自动刷新**：实现token预刷新机制，避免用户感知到token过期
3. **批量操作**：提供批量登出功能，提高会话管理效率
4. **连接池**：数据库和Redis连接使用连接池管理
