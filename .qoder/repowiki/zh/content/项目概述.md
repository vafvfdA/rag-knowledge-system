## 核心功能与架构全景

派聪明项目是一个企业级AI知识库管理系统，其核心目标是利用检索增强生成（RAG）技术，帮助用户高效管理和利用知识库信息。系统支持多租户架构，允许用户通过自然语言查询知识库，并获得基于自身文档的AI生成响应。

系统主要功能包括：
- **知识库管理**：支持文档上传、解析、索引和权限管理
- **AI驱动的RAG实现**：通过语义分块、向量化和混合搜索，为大语言模型提供精准上下文
- **企业级多租户**：通过组织标签实现数据隔离和权限控制
- **实时通信**：采用WebSocket技术提供实时聊天交互体验

## 文档处理管道详解

文档处理是派聪明系统的核心流程之一，从文档上传到最终索引的完整管道涉及多个步骤和组件的协同工作。该流程采用异步处理模式，确保系统性能和可扩展性。最新版本实现了流式处理机制，有效解决了大文件处理时的内存溢出（OOM）问题。

### 文档上传与初步处理

当用户上传文档时，系统首先通过`UploadController`接收文件，进行基本验证后将其存储到MinIO对象存储中，并在MySQL数据库中记录元数据。随后，系统将文件处理任务发布到Kafka消息队列，实现异步解耦。

### 文档解析与文本提取

`FileProcessingConsumer`作为Kafka消费者，监听文件处理任务。当接收到任务时，它调用`ParseService`服务进行文档解析。该服务使用Apache Tika库支持多种文档格式（PDF、DOC、DOCX、TXT等）的文本内容提取。

### 流式处理与内存管理

为解决大文件处理时的内存溢出问题，系统实现了流式处理机制，采用"父文档-子切片"策略：

1. **流式解析**：`ParseService`使用Apache Tika的流式API，逐块读取文件内容，避免一次性加载整个文件到内存
2. **父子文档切分**：当累积的文本达到"父块"大小（默认1MB）时，触发分块处理
3. **内存监控**：系统实时监控内存使用率，当超过阈值（默认80%）时触发垃圾回收
4. **异常处理**：如果内存不足，系统会抛出异常并记录详细信息

```java
private void checkMemoryThreshold() {
    Runtime runtime = Runtime.getRuntime();
    long maxMemory = runtime.maxMemory();
    long totalMemory = runtime.totalMemory();
    long freeMemory = runtime.freeMemory();
    long usedMemory = totalMemory - freeMemory;
    
    double memoryUsage = (double) usedMemory / maxMemory;
    
    if (memoryUsage > maxMemoryThreshold) {
        logger.warn("内存使用率过高: {:.2f}%, 触发垃圾回收", memoryUsage * 100);
        System.gc();
        
        // 重新检查
        usedMemory = runtime.totalMemory() - runtime.freeMemory();
        memoryUsage = (double) usedMemory / maxMemory;
        
        if (memoryUsage > maxMemoryThreshold) {
            throw new RuntimeException("内存不足，无法处理大文件。当前使用率: " + 
                String.format("%.2f%%", memoryUsage * 100));
        }
    }
}
```

### 智能文本分块

`ParseService`中的`splitTextIntoChunksWithSemantics`方法实现了智能文本分割算法，该算法优先保持语义完整性：

1. **按段落分割**：首先将文本按段落（双换行符）分割
2. **长段落处理**：对超过设定大小的段落，按句子边界进一步分割
3. **超长句子处理**：对超过设定大小的句子，使用HanLP智能分词进行语义切割
4. **块大小控制**：确保每个文本块不超过预设的`chunkSize`（默认512字符）

这种分层分割策略既保持了文本的语义完整性，又确保了后续处理的效率和准确性。

```java
private List<String> splitTextIntoChunksWithSemantics(String text, int chunkSize) {
    List<String> chunks = new ArrayList<>();
    String[] paragraphs = text.split("\n\n+");
    StringBuilder currentChunk = new StringBuilder();

    for (String paragraph : paragraphs) {
        if (paragraph.length() > chunkSize) {
            // 处理长段落
            List<String> sentenceChunks = splitLongParagraph(paragraph, chunkSize);
            chunks.addAll(sentenceChunks);
        } else if (currentChunk.length() + paragraph.length() > chunkSize) {
            // 当前块已满，保存并开始新块
            if (currentChunk.length() > 0) {
                chunks.add(currentChunk.toString().trim());
            }
            currentChunk = new StringBuilder(paragraph);
        } else {
            // 添加到当前块
            if (currentChunk.length() > 0) {
                currentChunk.append("\n\n");
            }
            currentChunk.append(paragraph);
        }
    }

    if (currentChunk.length() > 0) {
        chunks.add(currentChunk.toString().trim());
    }

    return chunks;
}
```

### 混合搜索服务

`HybridSearchService`实现了关键词搜索和语义搜索的结合，提供更准确的检索结果。该服务首先执行Elasticsearch的混合查询，然后根据用户权限过滤结果。

```java
public List<SearchResult> searchWithPermission(String query, String userId, int size) {
    // 1. 执行混合搜索
    List<SearchResult> results = elasticsearchService.hybridSearch(query, size);
    
    // 2. 权限过滤
    List<SearchResult> filteredResults = results.stream()
        .filter(result -> hasAccessPermission(result, userId))
        .collect(Collectors.toList());
    
    return filteredResults;
}
```

### 组件协同机制

系统采用以下机制确保组件间的高效协同：
- **异步处理**：使用Kafka实现文档处理的异步化，提高系统响应速度
- **状态管理**：使用Redis缓存对话历史和会话状态，确保实时通信的连贯性
- **错误处理**：各服务组件都有完善的异常处理机制，确保系统稳定性
- **性能监控**：通过`LogUtils`进行性能监控，记录关键操作的执行时间
- **流式处理**：采用"父文档-子切片"策略，有效避免大文件处理时的内存溢出
