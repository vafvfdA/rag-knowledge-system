## 状态设计概览

应用状态模块（App Store）是前端应用的核心状态管理单元，负责管理应用级的全局状态，包括主题配置、布局模式、页面缓存等。该模块采用 Pinia 作为状态管理库，通过 `defineStore` 定义了一个名为 `useAppStore` 的 store。App Store 与其他 store 模块（如 Theme Store 和 Tab Store）紧密协作，共同维护应用的全局状态。

## 核心状态字段分析

App Store 定义了多个核心状态字段，用于管理应用的各种配置和状态。

### 主题与布局状态

- **themeDrawerVisible**: 控制主题抽屉的可见性，通过 `useBoolean` 钩子管理布尔值状态。
- **siderCollapse**: 控制侧边栏的折叠状态，同样通过 `useBoolean` 钩子管理。
- **mixSiderFixed**: 控制混合布局中侧边栏的固定状态，初始化时从本地存储读取值。

### 页面与内容状态

- **reloadFlag**: 用于触发页面重载的标志位，通过设置为 `false` 再 `true` 来实现页面刷新。
- **fullContent**: 控制是否全屏显示内容区域。
- **contentXScrollable**: 控制内容区域是否可横向滚动。

### 国际化与响应式状态

- **locale**: 当前应用的语言环境，从本地存储中读取默认值。
- **isMobile**: 响应式状态，通过 `useBreakpoints` 监听屏幕尺寸变化，判断当前是否为移动端布局。

## Actions 与用户交互响应

App Store 定义了一系列 actions 来响应用户的交互操作。

### 页面重载

`reloadPage` action 用于重新加载当前页面。它首先将 `reloadFlag` 设置为 `false`，等待指定的持续时间后，再将其设置为 `true`。如果主题配置中的缓存重置策略为 `refresh`，则还会调用路由模块的 `resetRouteCache` 方法来重置路由缓存。

```typescript
async function reloadPage(duration = 300) {
  setReloadFlag(false);

  const d = themeStore.page.animate ? duration : 40;

  await new Promise(resolve => {
    setTimeout(resolve, d);
  });

  setReloadFlag(true);

  if (themeStore.resetCacheStrategy === 'refresh') {
    routeStore.resetRouteCache();
  }
}
```

### 切换布局

当用户在移动端和桌面端之间切换时，App Store 会自动调整布局。在移动端，侧边栏会被折叠，并将主题布局设置为垂直模式。在桌面端，则会恢复之前备份的主题设置。

```typescript
watch(
  isMobile,
  newValue => {
    if (newValue) {
      localStg.set('backupThemeSettingBeforeIsMobile', {
        layout: themeStore.layout.mode,
        siderCollapse: siderCollapse.value
      });

      themeStore.setThemeLayout('vertical');
      setSiderCollapse(true);
    } else {
      const backup = localStg.get('backupThemeSettingBeforeIsMobile');

      if (backup) {
        nextTick(() => {
          themeStore.setThemeLayout(backup.layout);
          setSiderCollapse(backup.siderCollapse);

          localStg.remove('backupThemeSettingBeforeIsMobile');
        });
      }
    }
  },
  { immediate: true }
);
```

### 更新主题

App Store 通过监听 `locale` 的变化来更新文档标题、全局菜单和标签页的国际化文本，并设置 dayjs 的语言环境。

```typescript
watch(locale, () => {
  updateDocumentTitleByLocale();
  routeStore.updateGlobalMenusByLocale();
  tabStore.updateTabsByLocale();
  setDayjsLocale(locale.value);
});
```

## Getters 与派生状态计算

App Store 通过计算属性（computed properties）来派生出一些状态。

### 侧边栏状态

`isMobile` 计算属性通过 `useBreakpoints` 监听屏幕尺寸变化，判断当前是否为移动端布局。这个状态被用于自动调整侧边栏的折叠状态和主题布局。

### 页头显示

虽然 App Store 本身没有直接定义页头显示的计算属性，但它通过 `fullContent` 状态间接影响页头的显示。当 `fullContent` 为 `true` 时，内容区域会全屏显示，页头可能会被隐藏或调整。

## 状态读取与变更最佳实践

### dark-mode-container.vue 组件

`dark-mode-container.vue` 组件是一个简单的容器组件，用于根据主题模式切换背景和文本颜色。它通过 `inverted` 属性来控制是否使用反色主题。

```vue
<template>
  <div class="bg-container text-base-text transition-300" :class="{ 'bg-inverted text-#1f1f1f': inverted }">
    <slot></slot>
  </div>
</template>
```

### theme-drawer 组件

`theme-drawer` 组件是主题配置的主界面，它通过 `useAppStore` 获取 `themeDrawerVisible` 状态，并将其绑定到 `NDrawer` 组件的 `v-model:show` 上，实现抽屉的显示和隐藏控制。

```vue
<script setup lang="ts">
const appStore = useAppStore();
</script>

<template>
  <NDrawer v-model:show="appStore.themeDrawerVisible" display-directive="show" :width="360">
    <NDrawerContent :title="$t('theme.themeDrawerTitle')" :native-scrollbar="false" closable>
      <DarkMode />
      <LayoutMode />
      <ThemeColor />
      <PageFun />
      <template #footer>
        <ConfigOperation />
      </template>
    </NDrawerContent>
  </NDrawer>
</template>
```

#### 切换布局模式

`layout-mode.vue` 组件通过 `v-model:mode` 绑定到 `themeStore.layout.mode`，实现布局模式的双向绑定。当用户选择不同的布局模式时，`themeStore` 的 `setThemeLayout` action 会被调用，更新布局模式。

```vue
<LayoutModeCard v-model:mode="themeStore.layout.mode" :disabled="appStore.isMobile">
  <!-- 布局模式的可视化表示 -->
</LayoutModeCard>
```

#### 更新主题

`dark-mode.vue` 组件通过 `NTabs` 组件实现主题模式的切换。当用户选择不同的主题模式时，`handleSegmentChange` 函数会被调用，进而调用 `themeStore` 的 `setThemeScheme` action 来更新主题模式。

```vue
<NTabs
  :key="themeStore.themeScheme"
  type="segment"
  size="small"
  class="relative w-214px"
  :value="themeStore.themeScheme"
  @update:value="handleSegmentChange"
>
  <NTab v-for="(_, key) in themeSchemaRecord" :key="key" :name="key">
    <SvgIcon :icon="icons[key]" class="h-23px text-icon-small" />
  </NTab>
</NTabs>
```

## 模块间协作关系

App Store 与 Theme Store 和 Tab Store 之间存在紧密的协作关系。

### 与 Theme Store 的协作

App Store 通过 `useThemeStore` 引用 Theme Store，并在 `reloadPage` action 中根据 `themeStore.page.animate` 的值来决定页面重载的持续时间。此外，当 `isMobile` 状态变化时，App Store 会调用 `themeStore.setThemeLayout` 来调整主题布局。

### 与 Tab Store 的协作

App Store 通过 `useTabStore` 引用 Tab Store，并在 `locale` 变化时调用 `tabStore.updateTabsByLocale` 来更新标签页的国际化文本。

```typescript
watch(locale, () => {
  updateDocumentTitleByLocale();
  routeStore.updateGlobalMenusByLocale();
  tabStore.updateTabsByLocale();
  setDayjsLocale(locale.value);
});
```
