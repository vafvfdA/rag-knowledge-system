
## 联合类型与类型安全
`union-key.d.ts` 文件通过 TypeScript 的联合类型（Union Types）定义了一组字符串字面量类型，实现了类型安全的常量约束。

```typescript
declare namespace UnionKey {
  type ThemeScheme = 'light' | 'dark' | 'auto';
  type LoginModule = 'pwd-login' | 'code-login' | 'register' | 'reset-pwd' | 'bind-wechat';
  type ThemeLayoutMode = 'vertical' | 'horizontal' | 'vertical-mix' | 'horizontal-mix';
  // ...
}
```

这些类型在 `constants` 文件中被用作 `Record` 的键类型，确保只有预定义的字符串才能被使用。例如：

```typescript
export const themeSchemaRecord: Record<UnionKey.ThemeScheme, App.I18n.I18nKey> = {
  light: 'theme.themeSchema.light',
  dark: 'theme.themeSchema.dark',
  auto: 'theme.themeSchema.auto'
  // 如果添加 'custom': '...'，TS 会报错
};
```

这种设计有效防止了拼写错误和非法值的传入，提升了代码健壮性。

## 实际应用场景
### 表单校验
在登录、注册等表单中，直接引用 `REG_USER_NAME`、`REG_PWD` 等正则常量进行校验，确保规则统一。

```typescript
// 示例：使用 REG_PWD 校验密码
if (!REG_PWD.test(password)) {
  // 提示密码格式错误
}
```

### 状态判断
在主题切换逻辑中，通过 `themeSchemaRecord` 的键进行判断：

```typescript
function toggleTheme(mode: UnionKey.ThemeScheme) {
  if (mode === 'dark') {
    document.documentElement.classList.add('dark');
  } else {
    document.documentElement.classList.remove('dark');
  }
}
```

### UI 选项生成
通过 `transformRecordToOption` 函数将记录对象转换为下拉选项：

```typescript
export function transformRecordToOption<T extends Record<string, string>>(record: T) {
  return Object.entries(record).map(([value, label]) => ({
    value,
    label
  })) as CommonType.Option<keyof T, T[keyof T]>[];
}
```

该函数被广泛用于将 `themeSchemaRecord`、`yesOrNoRecord` 等转换为 `el-select` 等组件所需的 `options` 数组。

## 集中管理的优势
### 避免硬编码
所有常量集中定义，避免在代码中散落字符串或魔法数字，提升可维护性。

### 类型安全
通过 TypeScript 的类型系统（`Record`、`Union Types`）确保常量使用的正确性，编译期即可发现错误。

### 国际化支持
常量值使用 `App.I18n.I18nKey` 类型，直接映射到多语言文件，便于国际化。

### 多环境配置
常量可结合环境变量进行动态配置，适应不同部署环境。

### 便于复用
统一的常量和枚举可在多个模块间共享，减少重复代码。

## 结论
通过对 `constants` 和 `enum` 目录的系统梳理，可以看出该项目通过 TypeScript 的高级类型特性（联合类型、泛型、Record 映射类型）实现了高度类型安全的常量管理机制。结合 `transformRecordToOption` 等工具函数，不仅避免了硬编码错误，还提升了开发效率和代码可维护性。这种集中化、类型化的常量管理方式值得在大型前端项目中推广使用。
