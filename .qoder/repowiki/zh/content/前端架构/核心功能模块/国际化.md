## 语言包结构设计

项目采用模块化语言包设计，通过`zh-cn.ts`和`en-us.ts`两个文件分别定义中文和英文的翻译项。语言包结构遵循分层命名空间设计，主要包含以下模块：

- **system**: 系统级文本，如应用标题、更新提示等
- **common**: 通用操作文本，如新增、删除、确认等
- **request**: 请求相关提示信息
- **theme**: 主题配置相关的界面文本
- **route**: 路由名称的翻译
- **page**: 页面级文本，特别是登录页的详细文案
- **form**: 表单验证相关的提示信息
- **dropdown**: 下拉菜单选项
- **icon**: 图标提示文本
- **datatable**: 数据表格相关文本

语言包使用嵌套对象结构组织翻译项，形成清晰的层级关系。例如`common.add`表示通用操作中的"新增"文本。这种设计使得翻译项具有良好的可读性和可维护性。

```typescript
const local: App.I18n.Schema = {
  system: {
    title: '派聪明',
    updateTitle: '系统版本更新通知'
  },
  common: {
    add: '新增',
    delete: '删除'
  }
};
```

## i18n实例初始化流程

i18n实例在`index.ts`文件中进行初始化，使用`vue-i18n`库实现多语言支持。初始化流程包括语言检测、本地存储持久化和实例创建三个主要步骤。

初始化时首先从本地存储中获取用户上次选择的语言，如果不存在则使用默认语言`zh-CN`。设置`fallbackLocale`为`en`，当目标语言中缺少某个翻译项时，会自动回退到英文版本。

```typescript
import { createI18n } from 'vue-i18n';
import { localStg } from '@/utils/storage';
import messages from './locale';

const i18n = createI18n({
  locale: localStg.get('lang') || 'zh-CN',
  fallbackLocale: 'en',
  messages,
  legacy: false
});
```

`setupI18n`函数用于在Vue应用中注册i18n插件，`$t`函数作为全局翻译函数导出，方便在组件中使用。`setLocale`函数用于切换当前语言，并更新i18n实例的locale值。

```typescript
export function setupI18n(app: App) {
  app.use(i18n);
}

export const $t = i18n.global.t as App.I18n.$T;

export function setLocale(locale: App.I18n.LangType) {
  i18n.global.locale.value = locale;
}
```

## 日期格式本地化配置

日期格式的本地化通过`dayjs.ts`文件进行配置。项目使用`dayjs`库处理日期显示，并根据当前语言设置相应的本地化格式。

`setDayjsLocale`函数负责设置dayjs的本地化语言，它接收一个语言参数，默认为`zh-CN`。函数内部通过`localMap`映射表将应用语言代码转换为dayjs支持的语言代码（如`zh-CN`转为`zh-cn`）。

```typescript
import { locale } from 'dayjs';
import 'dayjs/locale/zh-cn';
import 'dayjs/locale/en';
import { localStg } from '@/utils/storage';

export function setDayjsLocale(lang: App.I18n.LangType = 'zh-CN') {
  const localMap = {
    'zh-CN': 'zh-cn',
    'en-US': 'en'
  } satisfies Record<App.I18n.LangType, string>;

  const l = lang || localStg.get('lang') || 'zh-CN';

  locale(localMap[l]);
}
```

该函数首先尝试使用传入的语言，如果未提供则从本地存储中获取，最后回退到默认的中文。通过这种方式，确保日期显示格式与用户选择的语言保持一致。

## Naive UI组件库国际化适配

项目使用`naive.ts`文件对Naive UI组件库进行国际化适配。该文件导出两个映射表：`naiveLocales`用于UI组件的文本翻译，`naiveDateLocales`用于日期选择器等组件的日期格式化。

```typescript
import { dateEnUS, dateZhCN, enUS, zhCN } from 'naive-ui';
import type { NDateLocale, NLocale } from 'naive-ui';

export const naiveLocales: Record<App.I18n.LangType, NLocale> = {
  'zh-CN': zhCN,
  'en-US': enUS
};

export const naiveDateLocales: Record<App.I18n.LangType, NDateLocale> = {
  'zh-CN': dateZhCN,
  'en-US': dateEnUS
};
```

通过这种方式，Naive UI组件库中的按钮、提示、日期选择器等组件的文本内容能够根据应用语言自动切换。例如，日期选择器中的"今天"、"确定"等按钮文本会随语言切换而变化。

## $t函数使用示例

`$t`函数是项目中进行文本翻译的主要方式，可以在Vue模板和JavaScript逻辑中使用。以下是在不同场景下的使用示例：

在Vue组件模板中使用`$t`函数：

```vue
<template>
  <NButton>{{ $t('common.add') }}</NButton>
  <span>{{ $t('common.confirmDelete') }}</span>
  <h3>{{ $t('common.lookForward') }}</h3>
</template>
```

在组件中动态获取翻译文本：

```typescript
const tooltipContent = computed(() => {
  if (!props.showTooltip) return '';
  return $t('icon.lang');
});
```

在搜索组件中根据菜单项的i18nKey获取翻译：

```typescript
const title = (menu.i18nKey ? $t(menu.i18nKey) : menu.label).toLocaleLowerCase();
```

`$t`函数支持嵌套键名，如`common.add`表示从`common`模块中获取`add`翻译项。这种设计使得翻译查找高效且直观。

## 动态语言切换机制

动态语言切换通过`lang-switch.vue`组件实现，该组件提供语言切换的UI界面和交互逻辑。组件接收当前语言和语言选项作为props，并在用户选择新语言时触发`changeLang`事件。

```vue
<script setup lang="ts">
interface Props {
  /** 当前语言 */
  lang: App.I18n.LangType;
  /** 语言选项 */
  langOptions: App.I18n.LangOption[];
  /** 显示提示 */
  showTooltip?: boolean;
}

const props = withDefaults(defineProps<Props>(), {
  showTooltip: true
});

function changeLang(lang: App.I18n.LangType) {
  emit('changeLang', lang);
}
</script>
```

当用户选择新语言时，父组件会调用`setLocale`函数更新i18n实例的locale值，同时将语言选择保存到本地存储中，实现持久化。由于Vue的响应式系统，i18n实例的locale值变化会自动触发所有使用`$t`函数的组件重新渲染，从而实现界面文本的即时更新。

语言切换流程如下：
1. 用户在语言切换组件中选择新语言
2. 组件触发`changeLang`事件
3. 父组件调用`setLocale`函数
4. 更新i18n实例的locale值
5. 将语言选择保存到本地存储
6. 所有使用`$t`函数的组件自动更新显示文本
