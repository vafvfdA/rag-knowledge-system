## 数据服务概览

PaiSmart项目采用微服务架构，集成了多种数据服务以支持其RAG（检索增强生成）系统的核心功能。系统在容器化环境中运行，通过`application-docker.yml`文件集中管理所有数据服务的连接参数、认证机制和性能调优配置。这些服务包括：

- **MySQL**：作为主关系型数据库，存储用户、会话、文件元数据等结构化信息。
- **Elasticsearch**：提供全文检索和向量搜索能力，支持混合搜索和权限过滤。
- **Redis**：用作高速缓存，存储会话状态和对话历史，提升系统响应速度。
- **Kafka**：作为消息队列，实现文件处理任务的异步解耦，确保系统稳定性和可扩展性。
- **MinIO**：作为对象存储服务，存储用户上传的原始文件和分片，提供高可用的文件存储解决方案。

这些数据服务通过Spring Boot的自动配置机制与应用集成，确保了配置的一致性和可维护性。

## MySQL配置

在容器化环境中，MySQL的连接配置通过`application-docker.yml`文件进行管理，确保了环境的隔离性和配置的灵活性。配置主要包括连接参数、JPA属性和连接池设置。

### 连接参数与认证

MySQL的连接参数定义了应用与数据库之间的通信方式，包括主机地址、端口、数据库名称、用户名和密码。这些参数通过环境变量注入，增强了安全性。

```yaml
spring:
  datasource:
    url: jdbc:mysql://${MYSQL_HOST:mysql}:${MYSQL_PORT:3306}/${MYSQL_DATABASE:smartpai}?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
    username: ${MYSQL_USER:root}
    password: ${MYSQL_PASSWORD:password}
    driver-class-name: com.mysql.cj.jdbc.Driver
```

- **主机与端口**：使用`MYSQL_HOST`和`MYSQL_PORT`环境变量，允许在不同环境中灵活配置。
- **数据库名称**：`MYSQL_DATABASE`指定数据库名称，默认为`smartpai`。
- **认证机制**：通过`MYSQL_USER`和`MYSQL_PASSWORD`进行身份验证，密码通过环境变量注入，避免硬编码。

### JPA与Hibernate配置

JPA（Java Persistence API）配置定义了实体与数据库表的映射关系，以及Hibernate的运行时行为。

```yaml
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.MySQL8Dialect
```

- **DDL自动更新**：`ddl-auto: update`模式在应用启动时自动更新数据库模式，适合开发和测试环境，但在生产环境中应谨慎使用。
- **SQL日志**：`show-sql: true`和`format_sql: true`启用SQL语句的输出和格式化，便于调试和性能分析。
- **方言设置**：`hibernate.dialect`指定为`MySQL8Dialect`，确保生成的SQL语句与MySQL 8兼容。

### 连接池配置

连接池配置优化了数据库连接的管理和复用，提升系统性能和资源利用率。

```yaml
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      connection-timeout: 20000
      idle-timeout: 300000
      max-lifetime: 1200000
      leak-detection-threshold: 60000
```

- **最大连接数**：`maximum-pool-size: 20`限制了连接池中最大连接数，防止数据库过载。
- **最小空闲连接**：`minimum-idle: 5`确保池中始终有5个空闲连接，减少连接创建的开销。
- **超时设置**：`connection-timeout`（20秒）、`idle-timeout`（5分钟）和`max-lifetime`（20分钟）定义了连接的生命周期，防止连接泄漏。
- **泄漏检测**：`leak-detection-threshold: 60000`（60秒）启用连接泄漏检测，帮助发现未正确关闭连接的问题。

## Elasticsearch配置

Elasticsearch是PaiSmart系统中实现高效搜索的核心组件，其配置涵盖了连接参数、索引设置和混合搜索策略。

### 连接与认证

Elasticsearch的连接配置确保了应用与搜索服务的安全通信。

```yaml
  elasticsearch:
    uris: ${ELASTICSEARCH_HOSTS:http://elasticsearch:9200}
    username: ${ELASTICSEARCH_USERNAME:}
    password: ${ELASTICSEARCH_PASSWORD:}
```

- **URIs**：`ELASTICSEARCH_HOSTS`环境变量指定Elasticsearch集群的地址，默认为`http://elasticsearch:9200`，适用于Docker容器间的通信。
- **认证**：通过`ELASTICSEARCH_USERNAME`和`ELASTICSEARCH_PASSWORD`进行基本认证，保护搜索服务免受未授权访问。

### 索引映射配置

Elasticsearch的索引映射定义了`knowledge_base`索引中各字段的数据类型和分析器，直接影响搜索质量和性能。

```json
{
  "mappings": {
    "properties": {
      "fileMd5": {
        "type": "keyword"
      },
      "chunkId": {
        "type": "integer"
      },
      "textContent": {
        "type": "text",
        "analyzer": "standard"
      },
      "vector": {
        "type": "dense_vector",
        "dims": 2048,
        "index": true,
        "similarity": "cosine"
      },
      "modelVersion": {
        "type": "keyword"
      },
      "userId": {
        "type": "keyword"
      },
      "orgTag": {
        "type": "keyword"
      },
      "isPublic": {
        "type": "boolean"
      }
    }
  }
}
```

- **文本字段**：`textContent`使用`text`类型和`standard`分析器，支持全文检索。
- **向量字段**：`vector`是`dense_vector`类型，维度为2048，启用索引并使用余弦相似度进行向量搜索。
- **关键字字段**：`fileMd5`、`userId`、`orgTag`等使用`keyword`类型，用于精确匹配和聚合。
- **布尔字段**：`isPublic`用于标识文档的公开状态。

### 索引初始化

`EsIndexInitializer.java`负责在应用启动时检查并创建`knowledge_base`索引，确保搜索服务的可用性。

```java
@Component
public class EsIndexInitializer implements CommandLineRunner {
    @Autowired
    private ElasticsearchClient esClient;

    @Value("classpath:es-mappings/knowledge_base.json")
    private org.springframework.core.io.Resource mappingResource;

    @Override
    public void run(String... args) throws Exception {
        BooleanResponse existsResponse = esClient.indices().exists(ExistsRequest.of(e -> e.index("knowledge_base")));
        if (!existsResponse.value()) {
            createIndex();
        }
    }

    private void createIndex() throws Exception {
        String mappingJson = new String(Files.readAllBytes(mappingResource.getFile().toPath()), StandardCharsets.UTF_8);
        CreateIndexRequest createIndexRequest = CreateIndexRequest.of(c -> c
                .index("knowledge_base")
                .withJson(new StringReader(mappingJson))
        );
        esClient.indices().create(createIndexRequest);
    }
}
```

## Redis配置

Redis在PaiSmart系统中扮演着高速缓存的角色，主要用于管理会话状态和对话历史，显著提升系统响应速度。

### 连接与认证

Redis的连接配置简单直接，通过环境变量注入连接信息。

```yaml
  redis:
    host: ${REDIS_HOST:redis}
    port: ${REDIS_PORT:6379}
    password: ${REDIS_PASSWORD:}
    timeout: 5000ms
```

- **主机与端口**：`REDIS_HOST`和`REDIS_PORT`指定Redis服务器地址，默认为Docker服务名`redis`和端口`6379`。
- **认证**：`REDIS_PASSWORD`用于认证，增强安全性。
- **超时**：`timeout: 5000ms`设置操作超时时间为5秒，防止长时间阻塞。

### 缓存策略

`RedisRepository.java`定义了具体的缓存操作，包括会话ID和对话历史的存取。

```java
@Repository
public class RedisRepository {
    private final RedisTemplate<String, Object> redisTemplate;
    private final ObjectMapper objectMapper;

    public String getCurrentConversationId(String userId) {
        return (String) redisTemplate.opsForValue().get("user:" + userId + ":current_conversation");
    }

    public List<Message> getConversationHistory(String conversationId) {
        String json = (String) redisTemplate.opsForValue().get("conversation:" + conversationId);
        return json == null ? new ArrayList<>() : objectMapper.readValue(json, objectMapper.getTypeFactory().constructCollectionType(List.class, Message.class));
    }

    public void saveConversationHistory(String conversationId, List<Message> messages) throws JsonProcessingException {
        redisTemplate.opsForValue().set("conversation:" + conversationId, objectMapper.writeValueAsString(messages), Duration.ofDays(7));
    }
}
```

- **键设计**：采用`user:{userId}:current_conversation`和`conversation:{conversationId}`的命名模式，清晰地组织缓存数据。
- **数据序列化**：使用`ObjectMapper`将`Message`对象序列化为JSON字符串存储。
- **过期策略**：对话历史设置7天的过期时间，自动清理旧数据，节省内存。

## Kafka配置

Kafka在PaiSmart系统中作为消息队列，实现了文件处理流程的异步化，提高了系统的稳定性和可扩展性。

### 连接与认证

Kafka的连接配置定义了生产者和消费者的连接参数。

```yaml
  kafka:
    bootstrap-servers: ${KAFKA_BOOTSTRAP_SERVERS:kafka:9092}
    consumer:
      group-id: file-processing-group
      auto-offset-reset: earliest
    producer:
      key-serializer: org.apache.kafka.common.serialization.StringSerializer
      value-serializer: org.springframework.kafka.support.serializer.JsonSerializer
```

- **Bootstrap服务器**：`KAFKA_BOOTSTRAP_SERVERS`指定Kafka集群地址，默认为`kafka:9092`。
- **消费者组**：`group-id`定义消费者组，确保消息被正确消费。
- **序列化器**：生产者使用`StringSerializer`和`JsonSerializer`，支持字符串键和JSON值的序列化。

### 主题配置

`KafkaConfig.java`定义了文件处理主题的名称和消费者组，实现了配置的集中管理。

```java
@Configuration
public class KafkaConfig {
    @Value("${kafka.topic.file-processing:file-processing}")
    private String fileProcessingTopic;

    @Value("${kafka.consumer.group.file-processing:file-processing-group}")
    private String fileProcessingGroupId;

    public String getFileProcessingTopic() {
        return fileProcessingTopic;
    }

    public String getFileProcessingGroupId() {
        return fileProcessingGroupId;
    }
}
```

### 消息消费流程

`FileProcessingConsumer.java`监听`file-processing`主题，处理文件解析和向量化任务。

```java
@Service
@Slf4j
public class FileProcessingConsumer {
    private final ParseService parseService;
    private final VectorizationService vectorizationService;

    @KafkaListener(topics = "#{kafkaConfig.getFileProcessingTopic()}", groupId = "#{kafkaConfig.getFileProcessingGroupId()}")
    public void processTask(FileProcessingTask task) {
        log.info("Received task: {}", task);
        try {
            InputStream fileStream = downloadFileFromStorage(task.getFilePath());
            parseService.parseAndSave(task.getFileMd5(), fileStream, task.getUserId(), task.getOrgTag(), task.isPublic());
            vectorizationService.vectorize(task.getFileMd5(), task.getUserId(), task.getOrgTag(), task.isPublic());
        } catch (Exception e) {
            log.error("Error processing task: {}", task, e);
            throw new RuntimeException("Error processing task", e);
        } finally {
            if (fileStream != null) {
                fileStream.close();
            }
        }
    }
}
```

- **异步处理**：文件处理任务被放入Kafka队列，由独立的消费者线程处理，避免阻塞主应用。
- **错误处理**：捕获异常并重新抛出，触发Kafka的重试机制或死信队列，确保任务不丢失。

### 文件上传流程

`UploadService.java`实现了分片上传、合并和预签名URL生成的完整流程。

```java
@Service
public class UploadService {
    @Autowired
    private MinioClient minioClient;

    public void uploadChunk(String fileMd5, int chunkIndex, long totalSize, String fileName, MultipartFile file, String orgTag, boolean isPublic, String userId) throws IOException {
        String storagePath = "chunks/" + fileMd5 + "/" + chunkIndex;
        PutObjectArgs putObjectArgs = PutObjectArgs.builder()
                .bucket("uploads")
                .object(storagePath)
                .stream(file.getInputStream(), file.getSize(), -1)
                .contentType(file.getContentType())
                .build();
        minioClient.putObject(putObjectArgs);
    }

    public String mergeChunks(String fileMd5, String fileName, String userId) {
        String mergedPath = "merged/" + fileName;
        List<ComposeSource> sources = partPaths.stream()
                .map(path -> ComposeSource.builder().bucket("uploads").object(path).build())
                .collect(Collectors.toList());
        minioClient.composeObject(
                ComposeObjectArgs.builder()
                        .bucket("uploads")
                        .object(mergedPath)
                        .sources(sources)
                        .build()
        );
        String presignedUrl = minioClient.getPresignedObjectUrl(
                GetPresignedObjectUrlArgs.builder()
                        .method(Method.GET)
                        .bucket("uploads")
                        .object(mergedPath)
                        .expiry(1, TimeUnit.HOURS)
                        .build()
        );
        return presignedUrl;
    }
}
```

- **分片上传**：文件被分割成多个分片，每个分片独立上传到`uploads`存储桶的`chunks/`目录。
- **合并**：所有分片上传完成后，使用`composeObject` API在服务端合并成完整文件，存储在`merged/`目录。
- **安全访问**：通过`getPresignedObjectUrl`生成有效期为1小时的预签名URL，确保文件访问的安全性。

## 混合搜索服务分析

`HybridSearchService.java`是PaiSmart系统中实现混合搜索的核心组件，结合了文本匹配和向量相似度搜索，并集成了权限过滤机制。

### 混合搜索策略

混合搜索采用两阶段召回策略，结合KNN向量搜索和BM25文本搜索，提升搜索结果的相关性。

```java
public List<SearchResult> searchWithPermission(String query, String userId, int topK) {
    final List<Float> queryVector = embedToVectorList(query);
    SearchResponse<EsDocument> response = esClient.search(s -> {
        s.index("knowledge_base");
        // 第一阶段：KNN召回
        int recallK = topK * 30;
        s.knn(kn -> kn
                .field("vector")
                .queryVector(queryVector)
                .k(recallK)
                .numCandidates(recallK)
        );
        // 必须命中关键词 + 权限过滤
        s.query(q -> q.bool(b -> b
                .must(mst -> mst.match(m -> m.field("textContent").query(query)))
                .filter(f -> f.bool(bf -> bf
                        .should(s1 -> s1.term(t -> t.field("userId").value(userDbId)))
                        .should(s2 -> s2.term(t -> t.field("public").value(true)))
                        .should(s3 -> {
                            if (userEffectiveTags.isEmpty()) {
                                return s3.matchNone(mn -> mn);
                            } else if (userEffectiveTags.size() == 1) {
                                return s3.term(t -> t.field("orgTag").value(userEffectiveTags.get(0)));
                            } else {
                                return s3.bool(inner -> {
                                    userEffectiveTags.forEach(tag -> inner.should(sh2 -> sh2.term(t -> t.field("orgTag").value(tag))));
                                    return inner;
                                });
                            }
                        })
                ))
        ));
        // 第二阶段：BM25重打分
        s.rescore(r -> r
                .windowSize(recallK)
                .query(rq -> rq
                        .queryWeight(0.2d)
                        .rescoreQueryWeight(1.0d)
                        .query(rqq -> rqq.match(m -> m
                                .field("textContent")
                                .query(query)
                                .operator(Operator.And)
                        ))
                )
        );
        s.size(topK);
        return s;
    }, EsDocument.class);
}
```

- **第一阶段（召回）**：使用KNN搜索从向量空间中召回`topK * 30`个候选文档。
- **第二阶段（重打分）**：在召回的候选集中，使用BM25算法对包含查询关键词的文档进行重打分，最终返回`topK`个结果。
- **权重分配**：`queryWeight: 0.2`保留部分向量分数，`rescoreQueryWeight: 1.0`让BM25主导最终排序。

### 权限过滤

搜索结果通过布尔查询的`filter`子句进行权限过滤，确保用户只能访问其有权限的文档。

```java
.filter(f -> f.bool(bf -> bf
    .should(s1 -> s1.term(t -> t.field("userId").value(userDbId))) // 用户自己的文档
    .should(s2 -> s2.term(t -> t.field("public").value(true)))     // 公开文档
    .should(s3 -> { /* 用户所属组织的文档 */ })                    // 组织标签
))
```

- **个人文档**：用户可以访问自己上传的文档。
- **公开文档**：用户可以访问标记为公开的文档。
- **组织文档**：用户可以访问其所属组织标签下的文档，支持组织层级关系。

### 故障恢复

服务实现了完善的故障恢复机制，当向量生成失败时，自动降级为纯文本搜索。

```java

```

## 数据服务最佳实践

为了确保PaiSmart系统的稳定性和高性能，以下是各数据服务的配置和使用最佳实践。

### 连接池配置

- **MySQL**：根据应用负载调整`maximum-pool-size`，避免连接过多导致数据库压力过大。设置合理的`idle-timeout`和`max-lifetime`，防止连接泄漏。
- **通用建议**：监控连接池的使用情况，确保`active-connections`和`idle-connections`在合理范围内。

### 超时设置

- **数据库**：`connection-timeout`应设置为略大于网络延迟，避免因短暂网络波动导致连接失败。
- **Redis**：`timeout`设置为5秒，平衡了响应速度和容错能力。
- **Kafka**：消费者`max-poll-interval-ms`应根据处理任务的耗时设置，避免因处理时间过长导致消费者被踢出组。

### 故障恢复

- **Elasticsearch**：实现搜索服务的降级策略，当向量搜索失败时，自动切换到纯文本搜索。
- **Kafka**：配置死信队列（DLQ），捕获无法处理的消息，便于后续分析和重试。
- **MinIO**：在文件上传过程中，检查分片是否已存在，避免重复上传，提高效率。

### 安全性

- **认证**：所有数据服务均应启用认证，避免使用默认凭据。
- **网络**：在生产环境中，使用TLS加密所有服务间的通信。
- **密钥管理**：数据库密码、API密钥等敏感信息应通过环境变量或密钥管理服务注入，避免硬编码。
