## 基础环境配置

### JDK 17 安装与配置
根据项目 `pom.xml` 文件中的配置，本项目要求使用 **JDK 17**。请确保已正确安装并配置环境变量。

**验证步骤：**
```bash
java -version
```
输出应包含 `java version "17"` 或更高版本。

### Node.js 18 与 pnpm 安装
根据前端 `package.json` 文件中的 `engines` 字段，项目要求 **Node.js >=18.20.0** 和 **pnpm >=8.7.0**。

**安装 pnpm：**
```bash
npm install -g pnpm
```

**验证版本：**
```bash
node --version
pnpm --version
```

## 依赖服务部署

### MySQL 8.0 配置
后端 `application.yml` 配置文件中指定了 MySQL 连接信息。

**配置项：**
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/PaiSmart?useSSL=false&serverTimezone=UTC&allowPublicKeyRetrieval=true
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
```

**部署步骤：**
1. 安装 MySQL 8.0。
2. 创建名为 `PaiSmart` 的数据库。
3. 使用 `root` 用户和密码 `123456` 进行连接。
4. 确保 MySQL 服务在 `3306` 端口上运行。

### Redis 7.0 配置
`application.yml` 文件中配置了 Redis 连接。

**配置项：**
```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
```

**部署步骤：**
1. 安装 Redis 7.0 或更高版本。
2. 启动 Redis 服务，确保其在 `6379` 端口监听。
3. 代码中通过 `RedisTemplate` 配置了 `String` 类型的键序列化器和 `JSON` 类型的值序列化器。

```java
@Bean
public RedisTemplate<String, Object> redisTemplate(RedisConnectionFactory connectionFactory) {
    RedisTemplate<String, Object> template = new RedisTemplate<>();
    template.setConnectionFactory(connectionFactory);
    template.setKeySerializer(new StringRedisSerializer());
    template.setValueSerializer(new GenericJackson2JsonRedisSerializer());
    return template;
}
```

### Kafka 3.2.1 配置
`application.yml` 文件中启用了 Kafka，并配置了生产者和消费者。

**配置项：**
```yaml
spring:
  kafka:
    enabled: true
    bootstrap-servers: 127.0.0.1:9092
    topic:
      file-processing: file-processing-topic1
      dlt: file-processing-dlt

      group-id: file-processing-group
```

**部署步骤：**
1. 安装 Kafka 3.2.1。
2. 启动 ZooKeeper 和 Kafka 服务。
3. 确保 `9092` 端口开放。
4. Kafka 用于处理文件上传后的异步任务。

### MinIO 8.5.12 配置
`application.yml` 文件中配置了 MinIO 文件存储。

**配置项：**
```yaml
minio:
  endpoint: http://localhost:9000
  accessKey: minioadmin
  secretKey: minioadmin
  bucketName: uploads
  publicUrl: http://localhost:9000
```

**部署步骤：**
1. 下载并运行 MinIO 服务。
   ```bash
   minio server /data
   ```
2. 访问 `http://localhost:9000`，使用 `minioadmin`/`minioadmin` 登录。
3. 创建名为 `uploads` 的存储桶。
4. 确保 `9000` 端口开放。

**代码集成：**
`MinioConfig` 类通过 `@Value` 注解注入配置，并创建 `MinioClient` Bean。

```java
@Configuration
public class MinioConfig {
    @Value("${minio.endpoint}")
    private String endpoint;
    @Value("${minio.accessKey}")
    private String accessKey;
    @Value("${minio.secretKey}")
    private String secretKey;

    @Bean
    public MinioClient minioClient() {
        return MinioClient.builder()
                .endpoint(endpoint)
                .credentials(accessKey, secretKey)
                .build();
    }
}
```

## 前端应用配置与启动

### vite.config.ts 配置分析
前端使用 Vite 构建，`vite.config.ts` 文件定义了开发服务器配置。

**关键配置：**
```typescript
server: {
  host: '0.0.0.0',
  port: 9527,
  open: true,
  proxy: createViteProxy(viteEnv, enableProxy)
}
```
- **开发服务器端口：** `9527`
- **自动打开浏览器：** `true`
- **代理配置：** 通过 `createViteProxy` 将 `/api` 请求代理到后端 `8081` 端口，解决跨域问题。

### 前端启动命令
1. 进入 `frontend` 目录：

2. 安装依赖：
   ```bash
   pnpm install
   ```

3. 启动开发服务器：
   ```bash
   pnpm dev
   ```

**可用脚本：**
- `pnpm dev`: 启动开发服务器

## 服务验证与常见问题

### 服务验证方法
1. **后端验证：**
   - 访问 `http://localhost:8081/actuator/health`，应返回 `{"status":"UP"}`。
   - 检查日志，确认无 `ERROR` 级别日志。

2. **前端验证：**
   - 浏览器访问 `http://localhost:9527`，应能正常加载页面。
   - 打开开发者工具，检查网络请求是否成功。

3. **首页验证：**
   - 浏览器访问 `http://localhost:5173`，应能正常显示。

4. **依赖服务验证：**
   - MySQL: `mysql -h localhost -u root -p`
   - Redis: `redis-cli ping` (应返回 `PONG`)
   - Elasticsearch: `curl http://localhost:9200`
   - MinIO: 访问 `http://localhost:9000`

### 常见问题排查

**1. 端口冲突：**
- **现象：** 启动失败，提示端口被占用。
- **解决方案：**
  - 使用 `lsof -i :端口号` (macOS/Linux) 或 `netstat -ano | findstr 端口号` (Windows) 查找占用进程。
  - 终止占用进程或修改 `application.yml` 或 `vite.config.ts` 中的端口号。

**2. 依赖服务启动失败：**
- **现象：** 后端启动时报错，如 `Connection refused`。
- **解决方案：**
  - 逐一检查 MySQL、Redis、Elasticsearch、Kafka、MinIO 是否已正确启动。
  - 检查 `application.yml` 中的连接地址、端口、用户名和密码是否正确。

**3. 数据库连接问题：**
- **现象：** `Cannot connect to database`。
- **解决方案：**
  - 确认 MySQL 服务已启动。
  - 确认 `PaiSmart` 数据库已创建。
  - 检查 `application.yml` 中的 `url`、`username` 和 `password`。

**4. Kafka 消费者组问题：**
- **现象：** 文件上传后未触发处理。
- **解决方案：**
  - 检查 Kafka 主题 `file-processing-topic1` 是否存在。
  - 确认消费者组 `file-processing-group` 的消费者已启动。

**5. MinIO 存储桶问题：**
- **现象：** 文件上传失败。
- **解决方案：**
  - 登录 MinIO 控制台，确认 `uploads` 存储桶已创建。
  - 检查 `accessKey` 和 `secretKey` 是否正确。
