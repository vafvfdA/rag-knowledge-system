## 简介

派聪明（PaiSmart）是一个企业级 AI 知识库管理系统，采用检索增强生成（RAG）技术，支持文档上传、语义解析、向量索引和自然语言查询。本指南旨在帮助新手开发者在本地环境中快速搭建并运行 PaiSmart 项目，涵盖 Java、Node.js、MySQL、Elasticsearch、Redis、Kafka 和 MinIO 的安装与配置，以及前后端的构建与启动流程。

## 前置环境与依赖安装

为确保 PaiSmart 项目顺利运行，请先安装以下依赖环境：

### 1. Java 17
PaiSmart 后端基于 Spring Boot 3.4.2，需使用 Java 17。

```bash
# 验证 Java 版本
java -version
```

### 2. Maven 3.8.6+
用于构建后端项目。

```bash
# 验证 Maven 版本
mvn -v
```

### 3. Node.js 18.20.0+ 与 pnpm 8.7.0+
前端使用 Vue 3 + Vite 构建，包管理工具为 pnpm。

```bash
# 安装 pnpm
npm install -g pnpm

# 验证版本
node -v
pnpm -v
```

### 4. MySQL 8.0
用于持久化存储用户、文档、组织等信息。

```bash
# 启动 MySQL 并创建数据库
mysql -u root -p
CREATE DATABASE PaiSmart CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 5. Redis 7.0.11
用于缓存用户会话、Token 和临时数据。

```bash
# 启动 Redis
redis-server
```

### 6. Elasticsearch 8.10.0
用于文档的语义搜索与关键词检索。

```bash
# 启动 Elasticsearch
elasticsearch
```

> **注意**：若使用 HTTPS 模式，请确保配置正确的证书或使用开发模式跳过验证。

### 7. Kafka 3.2.1
用于异步处理文档解析任务。

```bash
# 启动 Zookeeper 和 Kafka
zookeeper-server-start.sh config/zookeeper.properties
kafka-server-start.sh config/server.properties
```

### 8. MinIO 8.5.12
用于存储上传的原始文件。

```bash
# 启动 MinIO
minio server /data
```

> 默认访问地址：`http://localhost:9000`，账号密码均为 `minioadmin`。

## 后端服务配置与构建

### 1. 配置 application.yml

主配置文件位于 `src/main/resources/application.yml`，关键配置如下：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/PaiSmart?useSSL=false&serverTimezone=UTC
    username: root
    password: 123456
  data:
    redis:
      host: localhost
      port: 6379
  kafka:
    bootstrap-servers: 127.0.0.1:9092
minio:
  endpoint: http://localhost:9000
  accessKey: minioadmin
  secretKey: minioadmin
  bucketName: uploads
  publicUrl: http://localhost:9000
elasticsearch:
  host: localhost
  port: 9200
  scheme: http
  username: elastic
  password: zVLf2sb05Pnuk8toM+ws
file:
  parsing:
    chunk-size: 512
    buffer-size: 8192
    max-memory-threshold: 0.8
```

> 开发环境建议使用 `application-dev.yml`，其 `scheme` 为 `http`，便于调试。生产环境可使用 `application-docker.yml`，包含更严格的安全配置。

### 2. 流式解析与内存管理

PaiSmart 已实现流式处理大文件的能力，通过"父文档-子切片"策略避免内存溢出（OOM）。核心配置在 `application.yml` 的 `file.parsing` 部分：

- `chunk-size`: 每个文本块的最大字符数（默认512）
- `buffer-size`: 输入流缓冲区大小（默认8KB）
- `max-memory-threshold`: 内存使用率阈值（默认0.8，即80%）

当内存使用率超过阈值时，系统会自动触发垃圾回收，若仍无法释放足够内存，将抛出异常。

## 前端应用配置与启动

### 1. 前端环境变量配置

前端通过 `import.meta.env` 读取环境变量，主要配置在 `.env` 文件中（需手动创建）：

```env
# .env.development
VITE_BASE_URL=/
VITE_APP_TITLE=派聪明
VITE_SERVICE_BASE_URL=http://localhost:8081/api/v1
VITE_HTTP_PROXY=Y
VITE_PROXY_LOG=Y
VITE_AUTH_ROUTE_MODE=dynamic
```

### 2. 代理配置解析

前端使用 Vite 代理将 `/proxy-default` 请求转发至后端 API，配置位于 `frontend/build/config/proxy.ts`：

```ts
proxy[item.proxyPattern] = {
  target: item.baseURL,
  changeOrigin: true,
  rewrite: path => path.replace(new RegExp(`^${item.proxyPattern}`), '')
};
```

结合 `vite.config.ts` 中的 `createViteProxy`，实现开发环境下的跨域请求代理。

### 3. 安装依赖并启动前端

```bash
# 进入前端目录
cd frontend

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

> 前端默认启动在 `http://localhost:9527`，并自动打开浏览器。

## 端到端功能验证示例

### 1. 用户注册与登录
- 打开前端首页，点击“注册”
- 输入用户名、密码，完成注册
- 使用注册账号登录，系统自动创建默认组织

### 2. 上传文档
- 进入“知识库”页面
- 点击“上传文档”，选择一个 `.pdf` 或 `.docx` 文件
- 系统自动调用 Apache Tika 解析内容，并通过 Kafka 异步处理
- **新特性**：对于大文件，系统采用流式处理，将文档分块解析，避免内存溢出

### 3. 查询文档
- 在搜索框输入与文档内容相关的自然语言问题
- 系统调用豆包 Embedding 模型生成向量，在 Elasticsearch 中进行混合搜索
- 返回最相关的文本片段，并由 DeepSeek 模型生成回答

### 4. 查看结果
- 在聊天界面查看 AI 生成的响应
- 验证是否包含“(来源#编号: 文件名)”引用信息

此流程验证了从文件上传、解析、索引到语义检索的完整链路。

## 常见问题排查

### 1. 端口冲突
- **现象**：服务无法启动，提示端口被占用
- **解决**：
  - 修改 `application.yml` 中的 `server.port`
  - 修改 `vite.config.ts` 中的 `server.port`
  - 使用 `lsof -i :端口号` 查找并终止占用进程

### 2. 依赖缺失
- **现象**：`mvn clean install` 或 `pnpm install` 报错
- **解决**：
  - 检查 Maven 镜像源配置
  - 检查 pnpm 源：`pnpm set registry https://registry.npmmirror.com`

### 3. 数据库连接失败
- **现象**：启动时报 `Access denied for user`
- **解决**：
  - 确认 MySQL 用户名密码正确
  - 确认数据库已创建：`CREATE DATABASE PaiSmart`

### 4. MinIO 连接失败
- **现象**：上传文件失败
- **解决**：
  - 确认 MinIO 服务已启动
  - 确认 `minio.endpoint` 配置正确
  - 确认 `bucketName` 存在或设置自动创建

### 5. Elasticsearch 拒绝连接
- **现象**：`Connection refused`
- **解决**：
  - 确认 `elasticsearch.yml` 中 `network.host` 配置为 `0.0.0.0`
  - 若使用 HTTPS，确保证书配置正确或切换为 HTTP 模式

### 6. 内存不足（OOM）
- **现象**：上传大文件时服务崩溃或报错
- **解决**：
  - 检查 `application.yml` 中 `file.parsing.max-memory-threshold` 配置
  - 适当降低 `chunk-size` 值
  - 增加 JVM 堆内存：`-Xmx4g`（根据机器配置调整）
  - 确认流式处理已启用，避免一次性加载整个文件到内存
